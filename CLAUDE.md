## Project Overview

**Vampire Survivalz** is a Vampire Survivors-style 2D top-down survival/roguelike game built in Unity 6000.0.59f2.

## Unity Version & Requirements

- **Unity:** 6000.0.59f2 (required — do not upgrade without explicit intent)
- **Render Pipeline:** Universal Render Pipeline (URP) 17.0.4 with 2D renderer
- **Input:** New Input System v1.14.2
- **Target:** Windows 64-bit, 1920x1080

## Build & Development

There are no external build scripts. All building is done through Unity Editor or Visual Studio.

- Open project: Launch Unity Hub and open the `D:\Github\Vampire Survivalz` folder with Unity 6000.0.59f2
- Build: Unity Editor → File → Build Settings → StandaloneWindows64 → Build
- Tests: Unity Editor → Window → General → Test Runner (Unity Test Framework v1.6.0 is installed)
- IDE: Visual Studio or Rider (both configured via package manager)

The `.sln` and `.csproj` files are auto-generated by Unity — do not edit them manually.

## Architecture

### Tech Stack
- **Rendering:** URP 2D (Renderer2D.asset, UniversalRP.asset in `Assets/Settings/`)
- **Input:** `Assets/InputSystem_Actions.inputactions` — the central input map; use `InputAction` references rather than polling `Input.*` legacy API
- **Post-processing:** `Assets/DefaultVolumeProfile.asset`

### Intended Game Systems
This is a Vampire Survivors clone. Expected architecture:
- **GameManager** — singleton controlling game state (playing, paused, game over)
- **PlayerController** — movement via New Input System, stats (health, speed, etc.)
- **EnemySpawner / Wave System** — timer-based enemy spawning that scales over time
- **Weapon/Attack System** — auto-firing weapons with upgrades (likely component-based on the player)
- **XP & Leveling** — pickup collection, level-up UI with weapon/passive choices
- **PoolingManager** — object pooling for bullets, enemies (performance-critical)

### Conventions to Follow
- Place scripts under `Assets/Scripts/<SystemName>/` (e.g., `Assets/Scripts/Player/`, `Assets/Scripts/Enemies/`, `Assets/Scripts/Weapons/`)
- Prefabs under `Assets/Prefabs/<Category>/`
- Art/sprites under `Assets/Art/`, audio under `Assets/Audio/`
- Use `[SerializeField] private` fields rather than public fields for Inspector exposure
- Prefer ScriptableObjects for shared configuration data (weapon stats, enemy stats, upgrade definitions)
- Use Unity Events or C# events for decoupled system communication rather than direct cross-system calls
